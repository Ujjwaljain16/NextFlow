You are a senior frontend engineer integrating a Sidebar UI into an existing Next.js + React Flow project.

The backend and architecture are already complete.
You must implement ONLY the Sidebar UI layer cleanly.

---

## GOAL

Build a minimal, Krea-inspired Sidebar (Node Library) using clean components.

---

## REQUIREMENTS

### 1. Create SidebarItem Component

File:
`/components/sidebar/sidebar-item.tsx`

Implement a reusable component with:

* icon (image)
* label (text)
* active state
* hover state

Use this exact structure:

* flex layout
* gap: 8px (gap-2)
* padding: px-4 py-2
* height: h-9
* rounded-md
* text-sm

---

### 2. Styling Rules

Use Tailwind only.

Base styles:

* text color: muted gray
* hover: bg-zinc-800 + text-white
* active: bg-zinc-900 + text-white + subtle shadow

Icon:

* size: w-5 h-5
* shrink-0

Text:

* truncate
* no bold

---

### 3. Sidebar Container

File:
`/components/sidebar/sidebar.tsx`

Structure:

* vertical column
* scrollable
* fixed width (~240px)
* dark background (zinc-950 or similar)

---

### 4. Node Integration

Sidebar must dynamically render nodes from:

NodeRegistry

For each node:

* render SidebarItem
* label = node name
* icon = optional (fallback if missing)

---

### 5. Interaction

When SidebarItem is clicked:

* trigger addNode action from Zustand store
* create a new node in React Flow canvas

DO NOT:

* hardcode node list
* duplicate node definitions

---

### 6. State Management

Use Zustand store:

* selected node
* node addition logic

Sidebar must NOT hold business logic.

---

### 7. Clean Structure

Organize files:

/components/sidebar/
sidebar.tsx
sidebar-item.tsx

Keep components small and focused.

---

## IMPORTANT RULES

* DO NOT copy raw HTML from Krea
* DO NOT include unnecessary Tailwind classes
* DO NOT introduce new architecture
* DO NOT mix backend logic into UI
* Keep implementation minimal and clean

---

## OUTPUT EXPECTATION

* SidebarItem component
* Sidebar container
* Integration with NodeRegistry
* Integration with Zustand

---

Goal:
A clean, minimal Sidebar that matches Krea’s density and spacing, and integrates correctly with the existing workflow system.
